This project heavily depends on feedback from the users. While the core part of the scheduling information comes from the Bus Vannin schedules and route change notifications, that is not the only source of information used by this app. We also get information from the users themselves. So we really encourage feedback. This feedback may be technical bugs in the app but it may also be errors in the schedule or even information on, for example, bus stops that are unmarked or hard to find. There are various ways feedback can be provided:

  • For general comments and discussions, use the Google group: https://groups.google.com/g/isle-of-man-bus-route-app-users. If you are a tester and have early access then you will already be a member of this group.
  • For bug reports, please use the issue tracking on Github. Although anyone can open an issue here, we would appreciate it is normal software developer practice is followed, e.g. always check that there is not already open issue covering it, provide detail that would allow someone to reproduce the problem etc. The Github issue tracker is at https://github.com/westbury/iom-bus-planner/issues.
  • If the problem is with an actual route, use the "Report Problem" button within the app. This button can be found by scrolling to the bottom of the route details. If the problem is with the list of offered routes (rather than just one of the routes) then just select one of the routes. The advantage of reporting problems this way is that the details of the route (such as origin, destination, and time) are sent with the report. We can paste the report directly into a test case to reproduce the problem.
